# Random Coffee Bot

Telegram-бот для нетворкинга в сообществах: автоматически подбирает участникам
пары для коротких неформальных встреч («random coffee»), рассылает контакты
и собирает обратную связь. Учитывает предпочтительный формат встречи
(онлайн / офлайн / не важно) и город, для онлайн-встреч сразу выдаёт ссылку
на видеозвонок, умеет запускать раунды автоматически по расписанию.

## Возможности

- **Регистрация анкеты** — имя, «о себе», формат встречи, город (для офлайна).
- **Алгоритм матчинга** — рандомизированный жадный подбор пар с учётом
  истории прошлых раундов (повторные пары исключаются), формата и города.
- **Онлайн-встречи** — для онлайн-пар бот сразу присылает ссылку на
  видеокомнату публичного [Jitsi](https://jitsi.org) (бесплатно, без
  регистрации; сервер настраивается). Участники также могут созвониться
  напрямую в Telegram по кликабельному контакту.
- **Офлайн-встречи** — формируются только между участниками одного города.
- **Fallback** — если подходящего партнёра в нужном формате/городе нет,
  участник всё равно получает пару (онлайн-встреча), а не остаётся без раунда.
- **Кликабельный контакт** — имя партнёра приходит как inline-упоминание,
  открывающее его профиль (при наличии `@username`).
- **Переписка через бота (релей)** — участник пишет боту, а тот передаёт
  сообщение собеседнику. Работает для любой пары, даже без `@username`.
- **Управление статусом** — пауза/возобновление участия.
- **Редактирование анкеты** — изменение любого поля без повторной регистрации.
- **Обратная связь** — через 5 дней после раунда бот спрашивает, как прошла
  встреча; результат доступен администратору.
- **Автопланировщик** — раунды могут запускаться автоматически по расписанию
  (например, каждый понедельник в 10:00).
- **Админ-команды** — ручной запуск раунда, рассылка, статистика.

## Команды бота

### Участнику

| Команда    | Описание                                                        |
|------------|-----------------------------------------------------------------|
| `/start`   | Регистрация; при повторном вызове — просмотр профиля            |
| `/edit`    | Изменить анкету (имя, «о себе», город, формат встречи)          |
| `/profile` | Показать текущую анкету и статус                                |
| `/pause`   | Приостановить участие (не попадать в раунды)                    |
| `/resume`  | Возобновить участие                                             |
| `/help`    | Список команд                                                   |

### Администратору

Доступны только аккаунту, чей Telegram ID указан в `ADMIN_TELEGRAM_ID`.

| Команда              | Описание                                               |
|----------------------|--------------------------------------------------------|
| `/match`             | Запустить раунд: подобрать пары и разослать уведомления|
| `/broadcast <текст>` | Разослать сообщение всем активным участникам           |
| `/stats`             | Статистика: участники, активные, раунды, % обратной связи |

> Узнать свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).

## Как это работает

1. Участник проходит регистрацию (`/start`): имя → «о себе» → формат встречи.
   Если выбран офлайн или «не важно» — бот дополнительно спрашивает город.
2. Раунд запускается вручную (`/match`) или автоматически по расписанию
   (`MATCH_SCHEDULE`).
3. Алгоритм матчинга (`internal/bot/matchmaker.go`):
   - перемешивает активных участников (Фишер–Йетс);
   - **проход 1** — жадно подбирает пары, уважая формат и город и избегая
     повторов из последних 5 раундов;
   - **проход 2 (fallback)** — оставшихся объединяет в онлайн-встречи.
4. Каждому участнику пары приходит анкета партнёра, кликабельный контакт и —
   для онлайн-встреч — ссылка на видеозвонок.
5. Чтобы написать собеседнику, участник просто отправляет сообщение боту —
   тот пересылает его партнёру (работает и без `@username`).
6. Через 5 дней бот присылает запрос обратной связи.

## Технологический стек

- **Go 1.22** — язык реализации
- **PostgreSQL 16** — хранилище данных
- **[pgx v5](https://github.com/jackc/pgx)** — драйвер PostgreSQL
- **[go-telegram-bot-api v5](https://github.com/go-telegram-bot-api/telegram-bot-api)** — Telegram Bot API
- **[golang-migrate](https://github.com/golang-migrate/migrate)** — миграции БД
- **[cleanenv](https://github.com/ilyakaznacheev/cleanenv)** — конфигурация
- **Docker + Docker Compose** — развёртывание

## Структура проекта

```
random-coffee/
├── cmd/bot/main.go            # точка входа
├── internal/
│   ├── bot/                   # бизнес-логика
│   │   ├── bot.go             # диспетчер событий, отправка сообщений
│   │   ├── fsm.go             # состояния диалога и inline-клавиатуры
│   │   ├── user_handlers.go   # команды и текстовые сообщения участника
│   │   ├── callback.go        # обработка нажатий inline-кнопок
│   │   ├── admin_handlers.go  # админ-команды (/match, /broadcast, /stats)
│   │   ├── matchmaker.go      # алгоритм подбора пар
│   │   └── scheduler.go       # автопланировщик и отложенная обратная связь
│   ├── config/                # конфигурация из переменных окружения
│   ├── db/                    # слой доступа к данным (DAO) на pgx
│   └── model/                 # доменные структуры (User, Round, Match, Pair)
├── migrations/                # SQL-миграции golang-migrate
├── Dockerfile                 # multi-stage сборка образа
├── docker-compose.yml         # сервисы bot + PostgreSQL
└── .env.example               # шаблон конфигурации
```

## Переменные окружения

Конфигурация задаётся в файле `.env` (см. `.env.example`).

| Переменная           | Обязательна | Описание                                                        |
|----------------------|:-----------:|-----------------------------------------------------------------|
| `BOT_TOKEN`          |     да      | Токен бота от [@BotFather](https://t.me/BotFather)              |
| `ADMIN_TELEGRAM_ID`  |     да      | Telegram ID администратора (для админ-команд)                   |
| `MATCH_SCHEDULE`     |     нет     | Расписание автораунда `DOW HH:MM`, напр. `MON 10:00`. Без него автозапуск отключён |
| `VIDEO_CALL_BASE`    |     нет     | Сервер видеовстреч для онлайн-пар, напр. `https://meet.ffmuc.net/`. По умолчанию — публичный Jitsi |
| `POSTGRES_HOST`      |     нет     | Хост БД (по умолчанию `localhost`; в compose — `db`)           |
| `POSTGRES_PORT`      |     нет     | Порт БД (по умолчанию `5432`)                                   |
| `POSTGRES_DB`        |     да      | Имя базы данных                                                 |
| `POSTGRES_USER`      |     да      | Пользователь БД                                                 |
| `POSTGRES_PASSWORD`  |     да      | Пароль БД                                                       |

Допустимые значения `DOW`: `MON`, `TUE`, `WED`, `THU`, `FRI`, `SAT`, `SUN`.

## Структура базы данных

Схема создаётся автоматически при первом запуске (миграции из `migrations/`).

### Таблица `users` — участники

| Поле               | Тип              | Описание                                            |
|--------------------|------------------|-----------------------------------------------------|
| `telegram_id`      | `BIGINT` PK      | Идентификатор пользователя Telegram                 |
| `username`         | `TEXT`           | `@username` для ссылки (может отсутствовать)         |
| `name`             | `TEXT NOT NULL`  | Имя участника                                       |
| `about`            | `TEXT NOT NULL`  | Описание «о себе»                                   |
| `city`             | `TEXT NOT NULL`  | Город (для офлайн-пар; иначе пусто)                 |
| `preferred_format` | `meeting_format` | Формат встречи: `online` / `offline` / `any`        |
| `state`            | `TEXT NOT NULL`  | Состояние диалога (FSM)                             |
| `is_active`        | `BOOLEAN`        | Участвует в раундах; `false` — на паузе             |
| `registered_at`    | `TIMESTAMPTZ`    | Дата и время регистрации                           |

### Таблица `rounds` — раунды матчинга

| Поле                 | Тип           | Описание                              |
|----------------------|---------------|---------------------------------------|
| `id`                 | `SERIAL` PK   | Идентификатор раунда                  |
| `started_at`         | `TIMESTAMPTZ` | Дата и время запуска                  |
| `participants_count` | `INTEGER`     | Число участников в раунде             |

### Таблица `matches` — пары участников

| Поле          | Тип                              | Описание                          |
|---------------|----------------------------------|-----------------------------------|
| `id`          | `SERIAL` PK                      | Идентификатор встречи             |
| `round_id`    | `INTEGER` FK → `rounds.id`       | Раунд                             |
| `user1_id`    | `BIGINT` FK → `users.telegram_id`| Первый участник                   |
| `user2_id`    | `BIGINT` FK → `users.telegram_id`| Второй участник                   |
| `feedback_u1` | `TEXT`                           | Обратная связь первого участника  |
| `feedback_u2` | `TEXT`                           | Обратная связь второго участника  |
| `created_at`  | `TIMESTAMPTZ`                    | Дата и время создания             |

## Быстрый старт

Требуется Docker и Docker Compose.

```bash
git clone https://github.com/Nishiramirai/random-coffee.git
cd random-coffee

cp .env.example .env
# отредактируйте .env: укажите BOT_TOKEN и ADMIN_TELEGRAM_ID

docker compose up -d --build
docker compose logs -f bot
```

Бот применит миграции и начнёт принимать сообщения. Напишите ему `/start`.

Остановка: `docker compose down` (данные сохранятся), полное удаление вместе
с базой: `docker compose down -v`.
